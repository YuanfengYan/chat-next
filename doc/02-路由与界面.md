# 路由与界面

## 1. 路由表

| URL | 文件 | 类型 | 行为 |
| --- | --- | --- | --- |
| `/` | `src/app/(frontend)/page.tsx` | Server Component | 未登录跳转 `/auth`；已登录进入本地最近会话 |
| `/auth` | `src/app/(frontend)/auth/page.tsx` | Server + Client | 登录/注册，已登录用户返回首页 |
| `/chat/[chatId]` | `src/app/(frontend)/chat/[chatId]/page.tsx` | Server + Client | 验证会话并装配聊天工作台 |
| `/api/chat` | `src/app/api/chat/route.ts` | POST Route Handler | 鉴权后返回 AI SDK UIMessage 流 |
| `/api/auth/[...all]` | `src/app/api/auth/[...all]/route.ts` | GET/POST Route Handler | Better Auth 通配接口 |
| `/admin/login` | 后台登录页 | Server Component | 后台登录入口 |
| `/admin` | 后台仪表盘 | Server Component | 指标、最近工具与云端会话 |
| `/admin/tools/*` | 工具后台页面 | Server Component | 列表、权限、日志、运行配置 |
| `/admin/assistants/*` | 助手后台页面 | Server Component | Prompt、模型参数、工具及知识库绑定 |

## 2. 页面与业务边界

- 页面负责 `params`、`searchParams`、认证分流和模块组合。
- `/chat/[chatId]` 将异步 `params` 解包后，把 `sessionId` 和用户展示信息传给 `ChatShell`。
- `ChatShell` 及其子组件处理工作台布局；业务动作统一来自 `useChatController`。
- 根布局设置中文语言、Geist 字体、全局样式和主题 Provider。

## 3. Server 与 Client Component

需要浏览器 API 或交互状态的文件使用 `"use client"`，例如 `HomeLauncher`、认证表单、聊天控制器和聊天组件。认证查询、数据库查询、环境变量和供应商调用留在服务端。

首页采用两阶段恢复：服务端先验证登录，客户端再读取 `localStorage`，选择最近会话或生成 UUID 后 `router.replace`。这是因为服务端不能读取浏览器本地存储。

## 4. 后台路由保护

后台 console 布局调用 `requireAdminSession()`：

1. 未登录时重定向 `/admin/login`。
2. 已登录但无管理员权限时渲染无权限界面。
3. 管理员进入 `AdminShell`，子页面在服务端调用只读查询服务。

布局层保护用于统一体验；敏感读写仍应在各数据服务或接口处重复授权。当前后台只有读取页面，没有管理写接口。
