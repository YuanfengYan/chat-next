# 测试与扩展

## 1. 当前自动化测试

| 区域 | 已覆盖 |
| --- | --- |
| 本地 Repository | 保存、读取、列表、删除、损坏记录、无效信封、图片脱敏 |
| 消息 Renderer | Markdown、未知 Part 降级、联网搜索来源 |
| AI 基础模块 | 模型白名单、结构化/纯文本错误 |
| 联网搜索 | 请求鉴权、来源规范化、缺失密钥 |
| 图片 OCR | OCR 请求与文字位置规范化 |
| 认证校验 | 邮箱规范化、昵称/邮箱/密码边界 |

测试环境使用 Vitest、jsdom 和 Testing Library。执行 `pnpm test` 运行全部用例。

## 2. 建议补充的高价值测试

- `/api/chat`：未认证、超长请求、消息上限、非法模型、文件数量、MIME/Base64/魔数、上游失败和中止。
- `useChatController`：首次标题、停止后保存、错误后保存、流中禁止切换、删除当前会话。
- 认证：禁用用户、重复注册、GitHub 配置开关和管理员角色/邮箱白名单。
- 后台服务：DTO 字段最小化、空数据、数据库错误和权限拒绝。
- 响应式 UI：桌面侧栏、移动 Sheet、长消息、工具状态和附件预览。

## 3. 新增模型

1. 在 `src/lib/ai/models.ts` 增加公开定义。
2. 在 server-only Provider Registry 中增加供应商解析和密钥检查。
3. 明确模型是否支持工具、推理或多模态，并限制不兼容能力。
4. 补充目录白名单、API 校验和供应商失败测试。
5. 若改为数据库动态目录，应保留服务端允许列表与 DTO，不能直接信任数据库或客户端输入。

## 4. 新增消息 Part

1. 保持 AI SDK `UIMessage` 与 `parts` 结构。
2. 新增专用 Renderer，覆盖 loading、result、error 状态。
3. 对外部链接使用新窗口安全属性，对未知内容保留安全降级。
4. 确认 Repository 的 Zod passthrough、持久化体积和敏感数据清理策略。
5. 添加渲染测试。

## 5. 新增 Tool

1. 在 `src/lib/ai/tools` 创建 server-only 模块。
2. 使用 Zod 定义有限输入和供应商响应，设置超时、长度与数量上限。
3. 输出稳定领域 DTO，过滤秘密和无关原始字段。
4. 在 `createChatTools` 中显式加入白名单，并评估 `stepCountIs(4)` 是否足够。
5. 为成功、缺密钥、超时、非法响应和上游错误补测试。
6. 添加对应 Part Renderer；可点击来源必须安全打开。

## 6. 接入云会话

优先新增远程 `SessionRepository`，而不是在组件内散落 API 请求。服务端接口必须按当前用户过滤会话，并校验消息数量、Part 类型、内容长度和并发版本。迁移策略应允许把版本 1 的 localStorage 数据显式导入，且不得重复写入。

## 7. 后台写能力

后台新增写操作时，需要配套：细粒度权限检查、Zod 输入校验、事务、不可变版本发布、凭证加密、审计摘要、并发冲突处理和失败测试。Tool URL/模板执行还需要 SSRF 防护、域名白名单、响应体大小限制和严格超时。
