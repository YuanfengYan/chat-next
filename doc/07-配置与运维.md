# 配置与运维

## 1. 环境变量

| 变量 | 必需性 | 用途 |
| --- | --- | --- |
| `DATABASE_URL` | 必需 | PostgreSQL 连接；认证和后台启动依赖 |
| `BETTER_AUTH_SECRET` | 必需 | Better Auth 签名密钥 |
| `BETTER_AUTH_URL` | 必需 | 认证基础 URL，本地默认 `http://localhost:3000` |
| `DEEPSEEK_API_KEY` | 聊天必需 | DeepSeek 服务端调用 |
| `QIANFAN_API_KEY` | 联网搜索必需 | 百度千帆搜索 |
| `QIANFAN_SEARCH_MODEL` | 可选 | 千帆搜索模型 |
| `BAIDU_IMAGE_API_KEY` | OCR 必需 | 百度 OCR Client ID |
| `BAIDU_IMAGE_SECRET_KEY` | OCR 必需 | 百度 OCR Client Secret |
| `ADMIN_EMAILS` | 可选 | 早期后台管理员邮箱白名单 |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | 可选 | GitHub OAuth，必须成对配置 |
| `DATABASE_ENCRYPTION_KEY` | 动态工具凭据必需 | 严格的 32 字节 Base64 AES-256-GCM 密钥 |
| `TOOL_ALLOWED_HOSTS` | 动态工具生产必需 | 逗号分隔的远程工具域名，支持 `*.example.com` |
| `TOOL_ALLOW_LOCALHOST` | 可选 | 仅开发环境显式允许 localhost 工具测试，默认 false |

`.env.example` 只能保留空值或非敏感示例。当前示例中的 GitHub Client ID 应按部署方配置理解，Secret 仍必须为空且不得提交真实值。

更换 `DATABASE_ENCRYPTION_KEY` 前必须先完成凭据轮换，否则已有密文无法解密。生产环境动态工具端点必须先加入 `TOOL_ALLOWED_HOSTS`，运行时仍会继续校验 DNS 和实际连接地址。

## 2. 本地启动

```powershell
pnpm install
Copy-Item .env.example .env.local
pnpm db:validate
pnpm db:generate
pnpm db:migrate
pnpm dev
```

数据库不可用时，Prisma 初始化会失败，认证和所有后台页面无法正常工作；即使聊天历史保存在 localStorage，聊天页面仍要求数据库认证会话。

## 3. 数据库变更

```powershell
pnpm db:validate   # 校验 Schema
pnpm db:generate   # 生成 Prisma Client
pnpm db:migrate    # 创建并应用开发迁移
pnpm db:deploy     # 部署环境应用已有迁移
pnpm db:studio     # 数据浏览工具
```

生产部署先配置环境变量，再运行 `pnpm db:deploy`，不要在生产使用交互式 `db:migrate`。

## 4. 质量门禁

```powershell
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

`build` 会验证 Next.js 服务端/客户端边界及路由构建；外部供应商调用不应成为构建期副作用。

## 5. 部署检查

- 使用 HTTPS，设置生产 `BETTER_AUTH_URL` 和高强度随机 Secret。
- 为部署平台配置 `maxDuration` 可支持的函数时长；聊天代码请求最多 90 秒。
- 确认 PostgreSQL 连接数、迁移状态和备份策略。
- 校验 DeepSeek、千帆和百度 OCR 的额度、地域、超时及出站网络。
- 日志不得记录 Authorization、Cookie、图片 Base64、密码、OAuth token 或完整工具载荷。
- 为 `/api/chat` 增加平台级限流、请求观测和告警是生产化的重要补充。
