# AI 与工具

## 1. 模型与传输

前后端共享模型目录位于 `src/lib/ai/models.ts`。当前只允许 `deepseek-chat`。客户端通过 `DefaultChatTransport` 请求 `/api/chat`；服务端 `resolveModel()` 再次校验 ID，并使用只存在服务端的 `DEEPSEEK_API_KEY` 创建模型实例。

新增模型时必须同时考虑目录展示、服务端 Provider 解析、能力差异、测试以及请求校验，不能仅修改客户端选项。

## 2. `POST /api/chat`

### 请求主体

```ts
{
  id: string;              // 1..128
  modelId: string;         // 默认 deepseek-chat
  messages: UIMessage[];   // 1..100
}
```

接口处理顺序：

1. 使用 Better Auth 请求头读取会话，未登录返回 401。
2. 解析 JSON，并以序列化长度 18,000,000 字符作为总量上限。
3. 用 Zod 校验会话 ID、模型 ID和消息基本结构。
4. 只检查最后一条用户消息的文件 Part，最多 3 张。
5. 校验 MIME、data URL、Base64、4 MiB 单图上限及文件魔数。
6. 从送给模型的消息中移除原始文件，追加图片工具提示。
7. 调用 `streamText`，最多 4 步工具循环，并传播请求中止信号。
8. 返回 AI SDK UIMessage 数据流。

`maxDuration = 90` 为部署平台提供最长执行时间提示。API 错误采用 `{ code, message, retryable }`，常见状态为 400、401、413、502、503。

## 3. 工具白名单

每次请求由 `createChatTools(images)` 显式创建：

| Tool | 触发场景 | 外部服务 | 主要限制 |
| --- | --- | --- | --- |
| `webSearch` | 实时事实、新闻、价格或明确联网请求 | 百度千帆 AI Search | 查询 2..300 字；30 秒；最多 8 个来源 |
| `imageRecognition` | 读取上传图片文字和位置 | 百度通用文字识别 | 本次请求内图片；最多 3 张；30 秒 |

工具在服务端执行，模型只能访问声明的输入 Schema 和规范化输出，不能读取环境变量或任意图片。

## 4. 联网搜索

`searchWeb()` 使用 `QIANFAN_API_KEY` 调用千帆接口，支持 week、month、semiyear、year、any 时间范围。供应商响应经 Zod 校验后转换为：查询、截断到 12,000 字的答案、最多 8 个结构化来源和搜索时间。来源 snippet 最大 500 字，图片与视频引用被过滤。

## 5. 图片 OCR

图片先在 Route Handler 完成内容校验，再作为闭包参数传给工具。OCR 凭证换取的 access token 在服务端内存缓存，并提前 60 秒失效。输出保留图片序号、方向、语言、最多 200 行文字、位置、概率、顶点及最多 100 个段落。

当前是文字识别而非通用视觉理解；适合文字提取和定位，不应描述为图像语义分析。

## 6. 错误与安全

- 密钥只在 `server-only` 模块读取。
- 外部响应必须通过 Zod，不把供应商原始对象直接交给 UI。
- 请求中止与超时合并，避免连接无限占用。
- 原始图片不进入模型消息，只能被当前请求绑定的 OCR Tool 访问。
- UI 使用统一错误消息；当前流内部错误可能包含工具错误文本，后续生产强化可增加错误码映射和日志脱敏。

## 7. 动态 HTTP Tool

后台可维护版本化 HTTP Tool。只有工具已启用、版本已发布并绑定当前模型时，运行时才会使用 AI SDK `dynamicTool` 装载。服务端 tool router 负责输入 Schema 校验、受限模板渲染、凭据注入、HTTPS 请求、输出 Schema 校验和调用审计。

请求模板固定包含 `pathParams`、`query`、`headers` 和 `body`，仅支持 `{{input.field}}` 与 `{{credential.alias}}` 占位符。生产环境必须通过 `TOOL_ALLOWED_HOSTS` 配置允许域名；开发环境只有显式设置 `TOOL_ALLOW_LOCALHOST=true` 才允许本地测试服务。

动态工具响应必须是 JSON。调用日志仅保存脱敏、截断摘要，不保存 Authorization、Cookie、API Key 或完整远端载荷。

## 8. 自定义 SSE v1

模型内部仍使用 AI SDK UIMessage 流；HTTP 边界使用统一的版本化 SSE 包络，响应头为 `x-chat-stream-version: 1`。前端自定义 transport 解析包络并还原标准 `UIMessageChunk`，因此文本、reasoning、source 和 tool parts 仍由 `useChat` 管理。

协议当前不支持断线续传。服务端会独立消费一条 tee 流，以保证浏览器断开时 Generation 和 ToolInvocation 仍进入明确终态。
