# 聊天模块

## 1. 模块职责

```text
src/features/chat/
├─ domain/              会话、草稿、摘要和 SessionRepository 契约
├─ application/         useChatController 等交互编排
├─ client/
│  ├─ components/       工作台、输入框、消息列表和侧边栏
│  ├─ renderers/        UIMessage Part 渲染器
│  ├─ stores/           Zustand UI 状态
│  ├─ repositories/     localStorage 与云会话 HTTP 适配器
│  └─ transport/        AI SDK/SSE 浏览器传输
└─ server/
   ├─ handlers/         聊天与会话 HTTP 用例入口
   ├─ repositories/     Prisma 云会话实现
   ├─ services/         云端 Generation 和消息生命周期
   └─ validation/       请求与附件校验
```

## 2. 状态归属

- AI SDK `useChat` 独占当前完整消息和流状态。
- `useChatController` 协调发送、停止、会话切换、持久化和路由跳转。
- Zustand 只保存会话摘要、活动会话 ID 和侧边栏等轻量 UI 状态。
- 完整历史只能通过 `SessionRepository` 访问。

## 3. 持久化

本地实现使用 `deepchat:sessions` 版本化信封，并在保存前清除文件 data URL。云端实现通过 `/api/sessions` 访问服务端 Repository，使用 revision 做乐观并发控制。新增数据库或远程实现不得绕过 `SessionRepository`。

## 4. 服务端聊天流程

1. Route Handler 将请求交给 `handleChatRequest`。
2. handler 校验登录态、模型、Skill、消息数量和请求大小。
3. validation 校验图片 MIME、base64、大小和文件魔数。
4. 云端模式由 generation service 校验会话归属、保存用户消息并创建 Generation。
5. AI orchestration 选择模型、加载 Skill 和 Tool，并产生 UIMessage 流。
6. generation service 保存助手消息，记录成功、取消或失败状态。
7. streaming 模块封装稳定 SSE 协议并返回客户端。

## 5. 消息渲染

消息始终按 `UIMessage.parts` 分派。文本、联网搜索和图片识别使用专用 Renderer，未知 Part 显示安全降级界面。Renderer 只依赖 AI domain 中的稳定工具结果类型，不得从 `.server.ts` 文件导入类型。
