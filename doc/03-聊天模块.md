# 聊天模块

## 1. 模块职责

`src/features/chat` 包含：

- `components/`：工作台、侧栏、工具栏、输入框、消息列表与消息项。
- `hooks/use-chat-controller.ts`：聊天业务编排中心。
- `store/chat-store.ts`：跨组件共享的轻量 UI 状态。
- `repositories/`：会话持久化实现。
- `renderers/`：按 `UIMessage.parts` 渲染文本、工具结果和未知类型。
- `types/chat.ts`：会话、草稿、附件及 Repository 契约。

## 2. 会话领域对象

`ChatSession` 保存 `id`、标题、模型 ID、完整 `UIMessage[]` 和时间戳；`SessionSummary` 去掉消息正文用于侧栏；`ChatDraft` 同时包含文本和待上传附件。

新会话默认模型为 `deepseek-chat`，标题由首条用户文本规范化并截取 28 个字符；空文本附件消息保留“新对话”标题。

## 3. 控制器流程

`useChatController(sessionId)` 统一协调：

1. 从 Repository 加载会话；不存在时创建并保存。
2. 用历史消息初始化 AI SDK `useChat`。
3. 刷新侧栏摘要并记录活动会话。
4. 发送时将 `File` 转为 data URL，并把当前模型放入请求 body。
5. 首次发送时更新标题。
6. 流结束时保存；失败或用户停止时延迟保存当前消息快照。
7. 切换、创建和删除会话时执行对应客户端路由跳转。

流正在 `submitted` 或 `streaming` 时禁止切换会话，避免流结果写入错误会话。

## 4. 状态归属

| 状态 | 所有者 | 原因 |
| --- | --- | --- |
| 当前完整消息和流状态 | AI SDK `useChat` | 高频更新且与传输生命周期绑定 |
| 当前完整会话 | Controller 局部 state/ref | 支持持久化且不污染全局 Store |
| 会话摘要、活动 ID、侧栏开关 | Zustand | 多个聊天组件共享 |
| 历史会话 | `SessionRepository` | 隔离存储方案，便于未来云同步 |

## 5. 本地持久化

存储键为 `deepchat:sessions`，外层格式为 `{ version: 1, sessions: [...] }`。读取时 Zod 校验信封及每条会话，损坏记录被隔离，整体格式无效时安全返回空列表。

保存前会把文件 Part 的 data URL 替换为 `attachment:expired`，避免在本地长期保留大体积、可能敏感的图片内容。因此刷新后附件只能降级展示元数据，不能重新识别。

## 6. 消息渲染

消息统一按 `UIMessage.parts` 分派：文本使用 Markdown/GFM；`tool-webSearch` 展示搜索过程、答案和来源卡片；`tool-imageRecognition` 展示 OCR 状态与识别结果；未知 Part 返回安全降级界面。新增 Part 类型时应新增专用 Renderer，并保留默认分支。

## 7. 当前限制

- 历史仅存在当前浏览器，清理站点数据后丢失，且不会跨设备同步。
- 多标签页没有同步或冲突合并机制。
- localStorage 有容量限制，大量长会话可能保存失败；当前写入失败未提供专门 UI。
- 数据库 `ChatSession`/`ChatMessage` 尚未接入前台会话仓库。
